# 📄 เอกสารทางเทคนิค: ระบบข้อมูลโทรมาตรอัตโนมัติ (Telemetry Dashboard)

เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายภาพรวม สถาปัตยกรรม และเทคนิคที่ใช้ในโปรเจกต์ **Telemetry Dashboard** เพื่อเป็นแนวทางสำหรับนักพัฒนาที่ต้องการนำไปศึกษาหรือต่อยอด

---

## 🌟 ภาพรวมของระบบ (Project Overview)
ระบบนี้ถูกออกแบบมาเพื่อเป็น **Dashboard แสดงผลข้อมูลโทรมาตรแบบ Real-time** สำหรับตรวจวัดระดับน้ำและปริมาณน้ำฝน โดยเน้นการแสดงผลผ่านแผนที่เชิงพื้นที่ (Web Map) และมีระบบหลังบ้านสำหรับบริหารจัดการข้อมูลที่ง่ายต่อการใช้งาน

### ความสามารถหลัก:
*   **Public Map**: แสดงพิกัดสถานีวัดน้ำบนแผนที่แยกตามสีสถานะ (ปกติ/เฝ้าระวัง/วิกฤต)
*   **Stackable Layers**: ระบบแผนที่ที่สามารถเลือกซ้อนเลเยอร์ได้หลากหลาย (ดาวเทียม, ถนน, แหล่งน้ำ)
*   **Detailed Analytics**: แสดงกราฟระดับน้ำย้อนหลัง (Simulated) และมาตรวัด (Gauge)
*   **Admin Management**: ระบบ CRUD จัดการสถานีและบัญชีผู้ใช้งานผ่าน UI ที่ทันสมัย
*   **Security**: ระบบล็อกอินและอนุมัติผู้ใช้รายบุคคล (Approved logic)

---

## 🛠️ Stack เทคโนโลยีที่ใช้ (Tech Stack)

### Frontend (Client-side)
*   **HTML5 / Vanilla JS**: ใช้การเขียน JavaScript แบบมาตรฐานเพื่อประสิทธิภาพสูงสุด
*   **Bootstrap 5**: ใช้ในการจัด Layout และส่วนประกอบ UI (Modals, Cards, Tabs)
*   **Leaflet.js**: หัวใจหลักในการจัดการแผนที่และชั้นข้อมูล GeoJSON
*   **Chart.js**: ใช้แสดงผลข้อมูลสถิติในรูปแบบกราฟเส้น
*   **SweetAlert2**: ใช้สำหรับการแจ้งเตือนที่สวยงามและ UX ที่ดี

### Backend (Server-side)
*   **Node.js & Express**: ใช้เป็น Web Server จัดการ Routes และ API
*   **PostgreSQL**: ระบบฐานข้อมูลเชิงสัมพันธ์ที่มีประสิทธิภาพสูงในการเก็บข้อมูลเชิงพื้นที่
*   **bcrypt**: ใช้เข้ารหัสผ่านเพื่อความปลอดภัย (Hashing)
*   **express-session**: จัดการ Session ของผู้ใช้

### Infrastructure
*   **Docker & Docker Compose**: ใช้รัน Environment ทั้งหมด (Web + DB) ให้พร้อมใช้งานได้ทุกเครื่อง

---

## 🚀 วิธีการรันระบบ (Getting Started)

1.  **ติดตั้ง Docker**: ตรวจสอบว่าเครื่องมี Docker Desktop ติดตั้งอยู่
2.  **สั่งรันระบบ**:
    ```bash
    cd webmap
    docker-compose up -d
    ```
3.  **เข้าใช้งาน**:
    *   หน้าบ้าน: `http://localhost:2025`
    *   หน้า Admin: `http://localhost:2025/admin`
    *   User/Pass เริ่มต้น: `admin` / `123456` (แก้ไขได้ใน `init.sql`)

---

## 📊 โครงสร้างฐานข้อมูล (Database Schema)
*   `stations`: เก็บพิกัด, ระดับน้ำ, ปริมาณฝน, และระดับการแจ้งเตือน
*   `users`: เก็บข้อมูลผู้ใช้, รหัสผ่านที่เข้ารหัสแล้ว, และสถานะการอนุมัติ (`approved`)
*   `activity_logs`: บันทึกประวัติการกระทำของแอดมิน (Audit Trail)
*   `api_configs`: ตั้งค่าสำหรับส่งข้อมูลต่อไปยัง API ภายนอก

---

## 💡 เทคนิคที่ควรรู้ (Key Techniques)
1.  **Layer Stacking**: การใช้ `L.control.layers` โดยแยก Base Maps ออกจาก Overlays ทำให้ผู้ใช้สามารถ "ติ๊กถูก" เพื่อเปิดข้อมูลหลายชั้นพร้อมกันได้
2.  **Dynamic Status Colors**: ระบบจะคำนวณสี Marker แบบ Dynamic ตามค่า `status` ในฐานข้อมูล ซึ่งแอดมินสามารถปรับค่าผ่านหลังบ้านได้ทันที
3.  **AJAX Workflow**: การล็อกอินและบันทึกข้อมูลใช้ `fetch()` ทั้งหมด ทำให้หน้าเว็บไม่ Refresh (Single Page Experience)

## 🚀 ข้อเสนอแนะในการต่อยอด (Future Roadmap)

### 1. ระบบแจ้งเตือนอัตโนมัติ (Alerting System)
*   **Line Notify**: เมื่อระดับน้ำในฐานข้อมูลอัปเดตเป็น `critical` ให้ส่งข้อความแจ้งเตือนพร้อมรูปแผนที่เข้ากลุ่มไลน์ทันที
*   **Email Alert**: ส่งรายงานสรุปสถานะสถานีที่ขัดข้องรายวัน

### 2. การเชื่อมต่อข้อมูลจริง (Hardware Integration)
*   **MQTT Protocol**: พัฒนา Endpoint รองรับการส่งข้อมูลจากกล่อง IoT หรือบอร์ด ESP32 โดยตรง
*   **Scraper**: เขียน Script ดึงข้อมูลจากเว็บหน่วยงานรัฐอื่นๆ มาพักไว้ในฐานข้อมูลตัวเอง

### 3. การวิเคราะห์ข้อมูล (Data Science)
*   **Trend Prediction**: ใช้สมการทางสถิติหรือ AI ช่วยทำนายแนวโน้มระดับน้ำในอีก 3-6 ชั่วโมงข้างหน้า
*   **Export Data**: เพิ่มปุ่ม Export ข้อมูลเป็น Excel หรือ CSV สำหรับทำรายงาน

### 4. ความปลอดภัย (Security Enhancement)
*   **Environment Variables**: ย้ายการตั้งค่า Database ไปไว้ในไฟล์ `.env`
*   **HTTPS**: ติดตั้ง SSL Certificate หากนำขึ้นเซิร์ฟเวอร์จริง
